▌ Тестовое задание: Сервис заказов с системой промокодов

▌ Стек технологий

- Python 3.12
- Django 5.x + Django Rest Framework
- PostgreSQL
- Poetry (управление зависимостями)
- Docker & Docker Compose

▌ Установка и запуск (через Docker)

▌ 1. Подготовка окружения

Переименуйте файл настроек .env.example в .env. Затем заполните необходимые переменные среды, включая настройки базы данных и секретные ключи.

cp .env.example .env

▌ 2. Запуск проекта

Запустите контейнеры приложения командой:

docker-compose up --build

Сервис станет доступным по адресу: http://localhost:8000

⎯⎯⎯⎯⎯⎯⎯⎯⎯⎯
▌ Локальная разработка (без Docker)

Если у вас установлен менеджер зависимостей Poetry, выполните шаги ниже:

▌ 1. Установка зависимостей

Установите требуемые пакеты с помощью команды:

poetry install

▌ 2. Активация виртуального окружения

Активируйте виртуальное окружение следующим образом:

poetry shell

▌ 3. Выполнение миграций БД

Выполните миграционные скрипты для создания таблиц базы данных:

python manage.py migrate

▌ 4. Запуск сервера разработки

Поднимите локальный веб-сервер командой:

python manage.py runserver

Проект запустится по адресу: http://127.0.0.1:8000

⎯⎯⎯⎯⎯⎯⎯⎯⎯⎯
▌ Запуск тестов

Для запуска модульных тестов выполните команду:

python manage.py test src.apps.orders

⎯⎯⎯⎯⎯⎯⎯⎯⎯⎯
▌ Ручное тестирование API

Чтобы вручную создать заказ, откройте страницу API-интерфейса:

http://127.0.0.1:8000/api/orders/create

Отправьте POST-запрос с JSON-данными следующего формата:

{
  "user_id": 1,
  "goods": [
    {
      "good_id": 1,
      "quantity": 1
    }
  ],
  "promo_code": ""
}

- Поле "user_id" — идентификатор пользователя.
- Поле "goods" — список товаров, включающих идентификаторы продуктов (good_id) и количество каждого товара (quantity).
- Поле "promo_code" — код промоакции (может быть пустым).
